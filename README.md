## Git Workflow

- Create your own branch.
- Work only on your assigned task.
- Commit your changes regularly.
- Push your branch to GitHub.
- Open a Pull Request.
- Wait for the Team Lead to review and merge your Pull Request.
- Pls when you push your work using the normal pushing we know, then generate a pull request (i'll drop the code or you could check it up), then i will merge it to the main branch myself so that it is consiste

## Collaboration Rules

- Do not work directly on the `main` branch.
- Create your own branch before starting work.
- Keep commit messages meaningful.
- Do not merge your own Pull Requests.
- Pull the latest changes from `main` before starting a new task.
